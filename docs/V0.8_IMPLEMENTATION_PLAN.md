# v0.8.0 Implementation Plan — Based on 2026 Research

## Research Date: 2026-06-21
## Sources: LangGraph, OpenAI SDK, Anthropic SDK, Microsoft GraphRAG, HyperAgents, DSPy, Reflexion

---

## 1. Real Anthropic SSE Streaming
**Pattern**: Raw fetch + TextDecoderStream + event state machine
**Events**: message_start → content_block_start → content_block_delta (text_delta|input_json_delta) → content_block_stop → message_delta → message_stop
**Key**: Accumulate tool_use blocks from input_json_delta chunks, parse JSON on content_block_stop
**Library**: Native fetch (no SDK needed), or `@anthropic-ai/sdk` for high-level API

## 2. SLM Router
**Pattern**: Heuristic complexity score (<1ms) → embedding disambiguation (~5ms) → cascade on low confidence
**Scoring**: Length + code blocks + reasoning keywords + multi-step indicators
**Models**: Haiku 4.5 (SLM), Sonnet 4.5 (frontier), Opus 4.8 (reasoning)
**Key**: RouteLLM (ICLR 2025) — router trained on one pair transfers when models are swapped

## 3. GraphRAG
**Pattern**: Entity extraction (LLM) → Knowledge Graph → Leiden community detection → Community summaries → Hybrid retrieval (local + global search)
**Local search**: Vector search entities → graph traversal neighbors → community reports → LLM synthesize
**Global search**: Map-reduce over community reports → rate relevance → top-K → LLM answer
**Library**: graphology (in-memory), neo4j-driver (production), microsoft/graphrag (Python indexer)

## 4. Real OpenTelemetry
**Packages**: @opentelemetry/sdk-node, @opentelemetry/exporter-trace-otlp-http, @opentelemetry/api
**Pattern**: tracing.ts imported first → BatchSpanProcessor → custom spans with performance.now() latency
**Attributes**: gen_ai.system, gen_ai.request.model, gen_ai.usage.input_tokens, llm.latency_ms
**Backend**: Jaeger (free local), Grafana Tempo (cloud)

## 5. Agent Factory
**Pattern**: AgentInstance (isolated context) → AgentFactory (templates + spawn) → Supervisor/Swarm/Graph orchestrator
**Supervisor**: Plan → executeParallel (worker pool) → synthesize
**Swarm**: Peer handoffs with transfer_to_* tools
**Graph**: StateGraph with conditional edges, parallel Send API
**Key lesson**: "More agents ≠ more intelligence" — use domain specialization, not redundancy

## 6. LLM-Based Context Summarization
**Pattern A**: Progressive (iterative) — summarize chunk-by-chunk, feed summary forward
**Pattern B**: Sliding window — keep last N messages verbatim, summarize older ones
**Pattern C**: Map-reduce — parallel chunk processing for long documents
**Model**: Use gpt-4o-mini or claude-haiku (cheap/fast) for summarization

## 7. LLM-Based Reflection
**Reflexion**: Multi-trial with persistent verbal memory (plan → execute → evaluate → reflect)
**Self-Refine**: Single-pass FEEDBACK → REFINE loop, converge on "APPROVED"
**CRITIC**: Tool-interactive verification (search, compute, logic checks)
**Storage**: ReflectionMemory with sliding window (last 5 per task key)

## 8. Self-Modifying Prompts
**DSPy**: Stanford's automated prompt optimization (use Python, wrap via API)
**Metaprompt**: Strong model writes prompts for weaker model
**A/B Testing**: Bayesian bandit router auto-shifts traffic toward winners
**Pattern**: Generate variants (meta-LLM) → evaluate (test set + LLM judge) → select best

## 9. Workflow Execution Engine
**Pattern**: State machine with typed state, pure node functions, conditional edges
**Features**: Retry with exponential backoff, parallel fan-out/fan-in, state persistence
**Library**: @langchain/langgraph (JS port of LangGraph)
**Persistence**: MemorySaver (dev), PostgresSaver (production)

## 10. CUA Browser
**Decision tree**: Static page? → fetch+cheerio | Need visual reasoning? → Claude CUA | Need form interaction? → Playwright MCP | Full automation? → Playwright
**Lightest**: fetch + cheerio (no JS rendering, fastest)
**AI-native**: Playwright MCP (accessibility tree, token-efficient, selector-free)
**Heaviest**: Claude Computer Use (screenshot-based, full visual reasoning)
